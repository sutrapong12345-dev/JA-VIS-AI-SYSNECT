# คู่มือผู้ใช้งาน J.A.R.V.I.S. — SYSNECT AI Agent

## 1. การเข้าสู่ระบบ

เปิด URL ของ J.A.R.V.I.S. ผ่าน Chrome หรือ Edge ระบบจะสร้าง Secure Session
ให้อัตโนมัติ Session จะผูกกับแท็บปัจจุบันและไม่สามารถใช้เปิดประวัติของผู้อื่นได้

สถานะผู้ใช้มีสามระดับ:

| บทบาท | ความสามารถหลัก |
|---|---|
| Staff | สนทนา, ค้นความรู้ที่อนุญาต, upload เอกสาร, reminder, ดูสถานะระบบทั่วไป |
| Manager | ความสามารถ Staff และเครื่องมือ/เอกสารระดับผู้จัดการ |
| Admin | จัดการความรู้ โมเดล ระบบความปลอดภัย audit และ action ที่ต้องอนุมัติ |

หากองค์กรเปิด Cloudflare Access ผู้ใช้ต้องยืนยันตัวตนด้วยอีเมลองค์กรก่อนเข้า
หน้า J.A.R.V.I.S. ชื่อและบทบาทจะถูกส่งจากระบบยืนยันตัวตนโดยอัตโนมัติ

## 2. การสนทนา

พิมพ์คำถามในช่องด้านล่างแล้วกดส่ง ตัวอย่าง:

- `สรุปขั้นตอนเปิด ticket ให้หน่อย`
- `นโยบายวันลาของบริษัทเป็นอย่างไร`
- `ช่วยเขียนสรุปประชุมจากไฟล์นี้`
- `ตรวจสถานะ CPU และ RAM`

คำตอบที่ใช้ฐานความรู้องค์กรควรมี citation รูปแบบ
`[KB:ชื่อไฟล์#chunk-N]` เพื่อให้ตรวจแหล่งข้อมูลได้

แถบ `JARVIS WORKSPACE` มีเครื่องมือหลัก:

- `Dashboard` กลับไปดูภาพรวม CPU, Sensor, AI Engine และ Environment
- `Panels` เปิดหรือซ่อนแผงสถานะด้านซ้ายและขวา
- `Holo` เปิดหรือปิด JARVIS Holographic Activity Projector
- ปุ่มลูกศรวน ตรวจ Backend และเชื่อมต่อใหม่
- `New` ล้างบริบทของ Session ปัจจุบันและเริ่มบทสนทนาใหม่ โดยระบบจะถามยืนยันก่อน
- ปุ่มส่งจะเปลี่ยนเป็นปุ่มหยุดขณะที่ AI กำลังตอบ

Hologram จะแสดงสถานะตามงานจริง เช่น `ANALYZE`, `AI ROUTING`, `WEB SEARCH`,
`LOCAL DATA`, `TOOL WORKER`, `DOCUMENT INGEST`, `VISION CORE`, `QUALITY GATE`
และ `COMPLETE` สีฟ้าหมายถึงกำลังประมวลผล สีทองหมายถึงกำลังฟัง สีเขียวหมายถึง
สำเร็จ และสีแดงหมายถึงถูกปฏิเสธหรือเกิดข้อผิดพลาด

ด้านบนของ Hologram มีเส้นทางงาน `INPUT → CONTEXT → AI CORE → ACTION → OUTPUT`
จุดสว่างคือขั้นตอนปัจจุบันและจุดที่ผ่านแล้วคือขั้นตอนที่ทำเสร็จ ผู้ใช้จึงเห็นได้ทันที
ว่า JARVIS กำลังรับคำสั่ง, อ่านบริบท, คิด, ใช้เครื่องมือ หรือส่งผลลัพธ์ ภาพตรงกลาง
จะเปลี่ยนตามงาน เช่น Radar สำหรับการค้นหา, Waveform สำหรับเสียง, Document Scan
สำหรับไฟล์, Tool Core สำหรับการใช้เครื่องมือ, เครื่องหมายถูกเมื่อสำเร็จ และ Warning
เมื่อเกิดปัญหา

กดปุ่มขยายที่มุมขวาบนของ Hologram เพื่อเข้า Cinematic Focus Mode และกดอีกครั้ง
หรือกด `Esc` เพื่อย่อกลับ การขยายมีผลเฉพาะการแสดงผล ไม่เพิ่มสิทธิ์และไม่เปลี่ยนงาน
ที่กำลังทำอยู่ ระบบจะลดการเคลื่อนไหวโดยอัตโนมัติเมื่ออุปกรณ์ตั้งค่า Reduced Motion

ใน Focus Mode สามารถลากด้วยเมาส์หรือนิ้วเพื่อหมุนวัตถุ ใช้ล้อเมาส์หรือปุ่ม `+`/`−`
เพื่อซูม และกดปุ่มเป้าเพื่อคืนมุมมองเริ่มต้น ผู้ใช้คีย์บอร์ดใช้ปุ่มลูกศรหมุน,
`+`/`−` ซูม และ `0` คืนมุมมองได้ แผง `PROJECTED INTELLIGENCE` แสดงชนิด
วัตถุ, แหล่งบริบท, ระดับสิทธิ์และ trace ส่วน `LIVE EVENT STREAM` แสดงสามเหตุการณ์
ล่าสุดของงานปัจจุบัน

ช่องพิมพ์รองรับหลายบรรทัด: กด `Enter` เพื่อส่ง และ `Shift+Enter` เพื่อขึ้นบรรทัดใหม่
ระบบแสดงจำนวนตัวอักษรและจำกัดสูงสุด 8,000 ตัวอักษร

ปุ่มรูปตาและคลิปบอร์ดเป็นฟังก์ชันระดับเครื่อง หากยังไม่ login Admin ระบบจะแสดงหน้าต่างเข้าสู่ระบบแทนการทำงาน

หากไม่มี citation ในคำตอบเกี่ยวกับข้อมูลบริษัท ไม่ควรนำข้อมูลนั้นไปใช้ตัดสินใจ
สำคัญจนกว่าจะตรวจสอบกับเอกสารต้นทาง

คำถามตรงเกี่ยวกับวันหรือเวลา เช่น `วันนี้วันอะไร` และ `ตอนนี้กี่โมง` จะตอบจาก
นาฬิกา Backend ในเขต `Asia/Bangkok` โดยตรง ไม่ใช้ความจำของโมเดล คำตอบจะแสดง
ทั้ง พ.ศ., ค.ศ., เวลาและเขตเวลา ส่วนข่าว ราคา ตำแหน่งบุคคล กฎหมาย ตารางเวลา
และเวอร์ชันซอฟต์แวร์เป็นข้อมูลที่เปลี่ยนได้ ระบบต้องตรวจแหล่งข้อมูลล่าสุดก่อนตอบ
หากยังตรวจไม่ได้ ระบบควรแจ้งว่ายังยืนยันข้อมูลปัจจุบันไม่ได้แทนการเดา

## 3. อัปโหลดเอกสารและรูปภาพ

กดปุ่มแนบไฟล์ รองรับ PDF, DOCX, TXT และรูปภาพทั่วไป

- เนื้อหาไฟล์จะถูกใช้ใน Session ปัจจุบัน
- ห้าม upload รหัสผ่าน, private key หรือข้อมูลที่ผู้ใช้ไม่มีสิทธิ์เปิดเผย
- ข้อมูลจากไฟล์ถือเป็นข้อมูลภายนอก ระบบจะไม่ยอมให้คำสั่งที่ฝังอยู่ในไฟล์
  ข้ามสิทธิ์ของผู้ใช้
- ไฟล์ขนาดเกินขีดจำกัดจะถูกปฏิเสธ

## 5. Structured Tools

J.A.R.V.I.S. ใช้เครื่องมือที่กำหนด schema ไว้ล่วงหน้า ไม่อนุญาตให้โมเดล
สร้าง PowerShell หรือคำสั่งใหม่เอง

เครื่องมืออ่านข้อมูล เช่นสถานะระบบ ทำงานผ่าน restricted worker แยกจาก backend
เครื่องมือที่มีผลกระทบ เช่นเปิดโปรแกรม ต้องแสดงรายการรออนุมัติก่อน

หากพบข้อความ `รายการรออนุมัติ`:

1. อ่านชื่อ Tool และ Arguments ให้ครบ
2. ตรวจว่าเป็นสิ่งที่คุณสั่งจริง
3. Admin พิมพ์ `[ยืนยันคำสั่ง: รหัสผ่าน]` ภายใน 2 นาที
4. หากไม่แน่ใจ ไม่ต้องยืนยัน รายการจะหมดอายุเอง

อย่าส่งรหัสผ่านในช่องแชททั่วไป ใช้เฉพาะข้อความยืนยันที่ระบบร้องขอเท่านั้น

## 6. การเข้า Admin Mode

เลือกเมนูโปรไฟล์มุมขวาบน แล้วเลือกเข้าสู่ระบบ ใส่ชื่อผู้ใช้และรหัสผ่าน Admin

Admin สามารถ:

- สอนและลบความรู้
- สร้าง training dataset
- ตรวจคะแนนและความพร้อมของ training dataset
- เปลี่ยน AI provider
- ดู audit และ security events
- สั่ง lockdown
- อนุมัติ structured actions

Admin Mode หมดอายุตามเวลาที่กำหนดและไม่ฟื้นกลับหลัง backend restart

## 7. ความหมายของข้อผิดพลาด

| รหัส/สถานะ | ความหมาย | วิธีแก้ |
|---|---|---|
| 401 | Session หมดอายุหรือ backend restart | Refresh หน้าเว็บ ระบบจะสร้าง Session ใหม่ |
| 403 | ไม่มีสิทธิ์หรือพยายามเข้าข้อมูลต่าง Session | Login ด้วยบทบาทที่ถูกต้อง |
| 413 | ไฟล์หรือ request ใหญ่เกินไป | ลดขนาดไฟล์ |
| 429 | เรียกใช้งานถี่เกินไป | รอตามเวลาที่ระบบแจ้ง |
| 503 / LOCKDOWN | ระบบป้องกันตัวจากเหตุผิดปกติ | ติดต่อ Admin และอย่าพยายามซ้ำ |

## 8. หลักการใช้งานอย่างปลอดภัย

- ตรวจ citation ก่อนใช้ข้อมูลบริษัท
- อย่าส่ง secret หรือข้อมูลส่วนบุคคลเกินความจำเป็น
- อย่ายืนยัน action ที่ไม่ได้เป็นผู้ร้องขอ
- แจ้ง Admin เมื่อเห็นคำตอบผิดสิทธิ์หรือข้อมูลของบุคคลอื่น
- Logout Admin เมื่อใช้งานเสร็จ
- AI ช่วยเตรียมและวิเคราะห์งาน แต่การตัดสินใจสำคัญยังต้องมีมนุษย์รับผิดชอบ
