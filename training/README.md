# ระบบเพิ่มคุณภาพและ Training AI ของ J.A.R.V.I.S.

ระบบแบ่งเป็น 3 ชั้น เพราะการเพิ่มข้อมูลจำนวนมากไม่ได้แปลว่า AI จะดีขึ้น หากข้อมูลมีคำตอบผิดหรืออ้างสิทธิ์เกินจริง โมเดลจะเรียนรู้ปัญหาเหล่านั้นซ้ำ

## ชั้นที่ 1: Knowledge RAG — เพิ่มข้อเท็จจริง

ใช้สำหรับข้อมูลบริษัท ขั้นตอนทำงาน นโยบาย และคู่มือ โดยเพิ่มเอกสารใน `backend/knowledge` หรือสอนผ่านแผง Admin

- มีผลทันที ไม่ต้องใช้ GPU
- แบ่งสิทธิ์ด้วยชื่อไฟล์ `staff__`, `manager__`, `admin__`
- คำตอบแสดง citation `[KB:filename#chunk-N]`
- ห้ามใส่รหัสผ่าน API key หรือข้อมูลลับลงในเอกสารความรู้

## ชั้นที่ 2: Quality-gated Dataset — สร้างข้อมูลฝึก

เปิดโปรไฟล์ด้านขวาบน เข้าสู่ระบบ Admin แล้วเลือก “แผงควบคุมแอดมิน”

1. กด “ตรวจคุณภาพชุดข้อมูลปัจจุบัน”
2. ตรวจคะแนน จำนวนผ่าน/คัดออก และเหตุผล
3. กด “สร้างชุดฝึกใหม่ผ่าน Quality Gate”
4. ระบบจะรวมตัวอย่างที่ทบทวนแล้วใน `curated_seed.json` กับ chat logs
5. ระบบตัดตัวอย่างที่มีปัญหาและสร้างไฟล์ต่อไปนี้

| ไฟล์ | หน้าที่ |
|---|---|
| `dataset_sysnect.jsonl` | ตัวอย่างที่ผ่านทั้งหมด |
| `dataset_train.jsonl` | ชุดสำหรับฝึก |
| `dataset_validation.jsonl` | ชุดวัดผลที่โมเดลไม่ได้ใช้เรียนโดยตรง |
| `quality_report.json` | คะแนนและเหตุผลที่คัดออก โดยไม่เก็บข้อความบทสนทนาในรายงาน |

ไฟล์ dataset และรายงานถูก `.gitignore` เพราะอาจมีบทสนทนาจริงขององค์กร

### เกณฑ์พร้อมฝึก

- ตัวอย่างผ่านอย่างน้อย 100 คู่
- คะแนนเฉลี่ยอย่างน้อย 80/100
- อัตราผ่านอย่างน้อย 80%
- ไม่มีแถว JSONL เสีย

Quality Gate ตรวจสิ่งต่อไปนี้:

- ข้อมูลซ้ำและคำถามเดิมที่เกิดจากการ retry
- ข้อความ error หรือ quota จาก provider
- การอ้างว่าผู้ใช้มี GOD MODE หรือสิทธิ์ที่ระบบไม่ได้ยืนยัน
- คำแนะนำให้ข้ามสิทธิ์หรือใช้ credential ของผู้อื่น
- ข้อมูลที่มีลักษณะเป็น secret
- ภาษาปะปนผิดปกติและบุคลิกไม่สม่ำเสมอ
- คำขอเสี่ยงที่คำตอบไม่ได้กำหนดขอบเขตความปลอดภัย

ตรวจจาก command line โดยไม่ต้องติดตั้ง GPU libraries:

```powershell
.\venv\Scripts\python.exe finetune.py --check-only
```

รหัสออก `0` หมายถึงพร้อมฝึก และ `2` หมายถึงยังไม่ผ่านเกณฑ์

## ชั้นที่ 3: LoRA Fine-tuning — ปรับสไตล์โมเดล Local

ใช้เมื่อชุดข้อมูลผ่าน Quality Gate แล้วเท่านั้น เหมาะสำหรับปรับน้ำเสียง รูปแบบคำตอบ และพฤติกรรมเฉพาะที่มีตัวอย่างคุณภาพจำนวนมาก ไม่เหมาะสำหรับเก็บข้อเท็จจริงที่เปลี่ยนบ่อย

ให้ใช้ environment แยกใน WSL/Conda ไม่ติดตั้ง training packages ปะปนกับ backend:

```bash
pip install unsloth datasets transformers trl peft accelerate bitsandbytes
python finetune.py --train
```

สคริปต์จะ:

1. ตรวจ Quality Gate ก่อนโหลดโมเดล
2. โหลด Llama 3.1 8B แบบ 4-bit
3. ฝึก LoRA จาก `dataset_train.jsonl`
4. วัด `eval_loss` จาก `dataset_validation.jsonl`
5. เก็บ adapter และ GGUF ใน `outputs/`

หากข้อมูลไม่ผ่าน สคริปต์จะหยุดอัตโนมัติ ตัวเลือก `--force` มีไว้สำหรับการทดลองที่ผู้ดูแลยอมรับความเสี่ยงแล้วเท่านั้น

## วิธีเพิ่มตัวอย่างคุณภาพ

เพิ่มรายการใน `curated_seed.json` รูปแบบ:

```json
{
  "user": "คำถามที่สมจริงหนึ่งเรื่อง",
  "assistant": "คำตอบที่ถูกต้อง ปลอดภัย กระชับ และไม่อ้างผลที่ยังไม่ได้ตรวจ"
}
```

หลักสำคัญคือหนึ่งตัวอย่างควรสอนพฤติกรรมเดียว มีคำตอบที่ตรวจสอบแล้ว และไม่ผูกกับข้อมูลลับหรือค่าที่เปลี่ยนตลอดเวลา

ห้ามใส่คำตอบวันที่ปัจจุบัน ข่าวล่าสุด ราคา หรือตำแหน่งบุคคลแบบค่าคงที่ในชุดฝึก
เพราะข้อมูลจะเก่าทันที ให้สอน “พฤติกรรมการใช้แหล่งเวลาหรือค้นข้อมูลสด” แทน
คำถามวัน–เวลาถูกตอบจาก Backend โดยตรง ส่วนข้อมูลเปลี่ยนแปลงอื่นต้องมีแหล่งข้อมูล
ล่าสุดหรือระบุว่ายังไม่ได้ตรวจสอบ
